Privacy boundary

A viewing key opens the wallet. Glasspane opens a receipt.

Sealed Seed, spending key, OVK, viewing key, balance, other transactions

These stay out of the room.

Opened One selected output: recipient, amount, memo, tx evidence

This is what the reviewer can verify.

Rejected Wrong OCK, wrong tx, or modified receipt

The verifier fails loudly instead of producing a false payout row.

Verification path

The room is assembled from evidence, not trust.

1 Issue receipt

Disclose the per-output OCK for the chosen shielded payment.

2 Recover output

gp-core recovers recipient, value, and memo from raw tx data.

3 Publish room

gp-room writes a verified report and optional accounting CSV.

cargo run -p gp-room -- examples/rooms/zechub-demo/room.json --csv payouts.csv --out verified-room.json
